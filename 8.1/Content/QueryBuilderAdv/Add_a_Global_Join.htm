Adding a Global Join

 System Administrator Only

A global join is permanent and exists from session to session in the Query Builder data tree structure. Be very careful when creating global joins. You can inadvertently create a global join that causes severe performance problems.

To add

  1. Open the Query Builder, and then click the Global Joins button.

This button only appears if you are logged on as Admin.

  1. In the Global Join Manager dialog box, click Add.
  2. In the Add Global Join dialog box, under Parent Table and Join Field, select the table and field for the global join.
  3. Under Child Table and Join Field, select the table and field to join to the parent table and field.
  4. Under Join Type, select the type of join.

The default is Left.

  1. Under Join Options:
  1. Select a Visibility option to specify how the join displays in the Query Builder.

The default is Allow.

  1. Select a Cascade Type to specify how records are deleted through the join.

The default is Ignore (Do not use in cascade).

  1. Click OK.
