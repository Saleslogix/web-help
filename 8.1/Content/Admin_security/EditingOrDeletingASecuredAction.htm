Editing or Deleting a Secured Action

The system administrator and users with the appropriate role can access these features.The system administrator and users with an administrator role can access these features.

If you rename or delete a secured action, the secured action is de-coupled from all elements it was assigned to in the Application Architect. Although the relationship no longer exists, the element is still secured until you manually remove the secured action from the Applied Security property of the control in the Application Architect. Tell me more...

If you want to remove security from the element, delete the secured action and manually remove the action from the Applied Security property of the control in the Application Architect.

Saleslogix recommends you save the information on the Secured Action Locations tab before renaming or deleting an action.

How Do I?

Edit a secured action

  1. Open the Secured Action Detail view for the action you want to edit.
  2. Use the Roles tab to add or remove the secured action from a role.
  3. Click Save.

Delete a secured action

  1. Open the Secured Action Detail view for the action you want to delete.
  2. Click Delete.
  3. Click OK to confirm.

Recover a secured action

If you accidentally delete (or rename) a secured action, you can create a new secured action with the same name which re-links the action to any controls that use the action in the Application Architect. If you manually removed the action from the Applied Security property on the control in the Application Architect, the action will not re-link to the control. The new secured action name must EXACTLY match the previous secured action name.
