I Cannot Delete a User

Error message: Other users report to this user. This user cannot be deleted.

Solution: Clear the User is a Manager check box in the User Detail view of the user you want to delete.

 

Error message: This user currently owns records which must be reassigned before being deleted, or the user may be a Team Owner, and therefore must be removed from the team before being deleted.

Solution: Run territory realignment.

  • If the message appears after running a territory realignment, Integrity Checker may need to be run to change the SECCODEID field in various tables to the correct value. This can be done using the Integrity Checker with the following tests enabled: “Account Summary with wrong owner (Set to Account owner),” “Contacts with the wrong owner (Set to Account owner),” and “Opportunities with the wrong owner (Set to Account owner).”
  • The previous message may also display when the user is the owner of a team. In this case, another user must be selected as the team owner.
  • Another cause for this message is that territory realignment failed in a previous run and this user still owns records in either the CONTACT, ACCOUNT, ACCOUNTSUMMARY, or OPPORTUNITY table. The following SQL syntax showing users and their respective SECCODEID’s may prove useful:

SELECT UI.USERID, UI.USERNAME, SR.SECCODEID, SC.SECCODEDESC

FROM SECRIGHTS SR, USERINFO UI, SECCODE SC

WHERE UI.USERID = SR.ACCESSID AND

SR.SECCODEID = SC.SECCODEID

ORDER BY ACCESSID
